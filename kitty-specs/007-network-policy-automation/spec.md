# Feature 007 — Network Policy Automation

**Author**: Ryan Loiselle — Developer / Architect  
**AI tool**: GitHub Copilot — AI pair programmer / code generation  
**Updated**: February 2026

---

## Overview

When a new `NetworkTestDefinition` is saved, the HelloNetworkWorld API automatically evaluates
whether the configured destination + port + protocol is already permitted by an existing egress
`NetworkPolicy` in the GitOps repository. If no matching policy exists, the API creates a branch
in `bcgov-c/tenant-gitops-be808f`, commits a new `NetworkPolicy` YAML for the egress rule,
and opens a pull request for review.

This ensures that network access is provisioned in a least-privilege manner (only opened when
needed) while keeping the configuration in the GitOps repo as the single source of truth.

---

## User Stories

- As a DevOps engineer, I want adding a network test to automatically trigger a PR for the
  needed egress NetworkPolicy so I don't need to manually write Kubernetes YAML.
- As a security reviewer, I want the network policy PR to require review before merging so
  I can validate the access request before it is applied.
- As a developer, I want to see a status message in the test configuration UI indicating
  whether the required network policy exists or if a PR has been raised.

## Requirements

### Functional Requirements

- **FR-007-01**: On `POST /api/network-tests` (new test creation), trigger a background check:
  1. Examine current egress NetworkPolicies in `charts/hnw-app/templates/networkpolicies.yaml`
     by fetching the file from the GitOps repo via GitHub API
  2. Determine if an egress policy already permits traffic to the destination CIDR on the given port/protocol
  3. If no policy exists, generate a new `NetworkPolicy` YAML block and open a PR
- **FR-007-02**: Generated PR:
  - Branch name: `feat/hnw-egress-{destination-slug}-{port}`
  - Commit message: `feat(network-policy): allow egress to {destination}:{port}/{protocol}`
  - PR title: `[HNW] Allow egress to {destination}:{port} for network test`
  - PR body: includes test name, destination, port, protocol, and a note asking for security review
  - PR target branch: `main` in `tenant-gitops-be808f`
- **FR-007-03**: Store the PR URL and `PolicyStatus` on the `NetworkTestDefinition`:
  `PolicyStatus`: `Unknown` | `Covered` | `PrPending` | `PrMerged` | `ManuallyConfigured`
- **FR-007-04**: `GET /api/network-tests/{id}` includes `policyStatus` and `policyPrUrl` fields
- **FR-007-05**: GitHub API credentials: `GITHUB_TOKEN` injected via environment variable
  (Kubernetes Secret, never in source code)
- **FR-007-06**: If `GITHUB_TOKEN` is not set, skip automation and set `PolicyStatus = Unknown`

### Non-Functional Requirements

- **NFR-007-01**: PR creation is async — does not block the create-test API response
- **NFR-007-02**: One PR per unique `(destination, port, protocol)` tuple — do not create duplicates
  if a test for the same destination already raised a PR
- **NFR-007-03**: CIDR resolution: if destination is an FQDN, do not attempt to resolve to CIDR;
  use the FQDN directly in the NetworkPolicy `ipBlock` → document that the operator must update
  the CIDR manually after DNS resolution

## Generated NetworkPolicy Shape

```yaml
# Generated by HelloNetworkWorld network policy automation
# Test: {testName}
# Date: {ISO date}
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: hnw-egress-{destination-slug}-{port}
  labels:
    app.kubernetes.io/managed-by: hello-network-world
    generated-for-test: "{testId}"
spec:
  podSelector:
    matchLabels:
      app.kubernetes.io/name: hnw-api
  policyTypes: [Egress]
  egress:
    - to:
        - ipBlock:
            cidr: {destinationCidr}/32   # Operator: replace with actual CIDR if FQDN
      ports:
        - port: {port}
          protocol: {TCP|UDP}
```

## Success Criteria

- [ ] Creating a test for a new destination triggers a GitHub PR in `tenant-gitops-be808f`
- [ ] Creating a second test to the same destination:port does not create a duplicate PR
- [ ] `GET /api/network-tests/{id}` returns `policyStatus: "PrPending"` and `policyPrUrl`
- [ ] Without `GITHUB_TOKEN`, test creates successfully with `policyStatus: "Unknown"`
- [ ] PR body contains destination, port, protocol, and security review note

---

## Out of Scope

- Auto-merging the PR (always requires human review)
- FQDN → CIDR resolution automation (operator must supply the CIDR block)
- Generating policies for non-HNW-API pods
